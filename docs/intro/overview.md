# プロジェクト概要

本リポジトリは、**Pythonアプリケーション向けの開発テンプレート**を提供するリポジトリです。
設計ドキュメント、リファレンス実装、開発ワークフローを提供し、新規プロジェクトの立ち上げを高速化します。

## 背景と目的

AI駆動開発ではLLMのメモリがセッションごとにリセットされるため、設計の一貫性維持・暗黙知の保存・新規プロジェクト立ち上げコストが課題となる。詳細は[設計コンセプト](concept.md)を参照。

| 課題 | テンプレートの対処 |
|------|-----------------|
| 設計の一貫性 | アーキテクチャ設計書（`docs/design/`）で判断基準を形式知化 |
| 暗黙知の喪失 | 仕様ドキュメント（`docs/specs/`）で設計意図を明示的に記述 |
| 立ち上げコスト | リファレンス実装・ワークフロー・ツール設定を即利用可能な形で提供 |

## テンプレートが提供するもの

### リファレンス実装（`src/example/`・`tests/`）

Onion Architecture と Composition Root + Orchestrator パターンを適用した、動作するサンプル実装。
新規機能パッケージを追加する際の規範となる。
またリポジトリ内のソースコードは AI への例示としても機能するため、AI が生成するコードの品質を高める目的でも徹底的にリファクタリングされている。

`src/example/` 配下のすべて（`cli.py`・`config/`・`foundation/`・`transform/`）がリファレンス実装であり、`tests/` 配下もテストコードのリファレンス実装として位置付ける。なかでも `transform/` パッケージは、アーキテクチャパターンの全要素を網羅した雛形として特に重要である。詳細は[設計コンセプト](concept.md)を参照。

### 設計ドキュメント（`docs/design/`）

プロジェクト全体に適用するアーキテクチャ設計・開発規約を記述したドキュメント群。

| ファイル | 内容 |
|---------|------|
| `architecture.md` | レイヤー構成、各種設計パターン、ガードレール |
| `specs.md` | 仕様ドキュメントの目的・設計思想・記述ガイドライン |
| `pyproject.toml` | ビルドシステム・リンター・型チェッカーの設定根拠 |

### 仕様ドキュメント（`docs/specs/`）

各モジュールの要件定義（`requirements.md`）と基本設計（`design.md`）をセットで管理するディレクトリ。
LLMがセッションをまたいでも設計思想に沿ったコード変更を行えるよう、明示的なコンテキストを提供する。

### 開発ワークフロー（Makefile, pyproject.toml）

すぐに使える開発コマンドとツール設定を提供する。

| コマンド | 内容 |
|---------|------|
| `make fmt` | コードフォーマット（ruff） |
| `make lint` | 静的解析（ruff） |
| `make typecheck` | 型チェック（pyright） |
| `make test-unit` | ユニットテスト（pytest） |
| `make coverage` | カバレッジ計測 |
